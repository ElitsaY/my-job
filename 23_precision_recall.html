Precision and Recall

Precision and recall are core evaluation metrics for ranked and unranked retrieval systems.

Overview

Precision asks: “How accurate are the retrieved results?”

Recall asks: “How complete are the retrieved results?”

1. Basic Setup

Suppose a collection has 100 documents. For a query:

  • 20 documents are truly relevant.
  • The system retrieves 15 documents.
  • Out of those 15, 12 are relevant.
Relevant Not Relevant
Retrieved123
Not Retrieved877

Where: TP = 12, FP = 3, FN = 8, TN = 77.

2. Precision

Precision = TP / (TP + FP)
Precision = 12 / 15 = 0.8

So 80% of retrieved documents are relevant.

3. Recall

Recall = TP / (TP + FN)
Recall = 12 / 20 = 0.6

So the system finds 60% of all relevant documents.

4. Intuition

Imagine searching for “research papers on neural networks”:

  • High precision → most results are actually about neural networks.
  • High recall → you found almost all relevant papers.

5. Precision vs Recall Tradeoff

  • Retrieve only top 3 highly relevant docs → precision high, recall low.
  • Retrieve 50 docs with weak matches → recall high, precision lower.

They usually move in opposite directions.

6. Extreme Cases

  • Retrieve nothing → recall = 0, precision undefined.
  • Retrieve everything → recall = 1, precision is very low.

7. Visual Interpretation

Think of two overlapping circles:

  • All relevant documents = one circle.
  • Retrieved documents = another circle.
  • Overlap = relevant retrieved.

Precision = overlap / retrieved

Recall = overlap / relevant

8. Which Is More Important?

Application More Important

9. F1 Score (Combined Measure)

F1 = 2 × (Precision × Recall) / (Precision + Recall)
F1 = 2 × (0.8 × 0.6) / (0.8 + 0.6) = 0.685

F1 balances precision and recall.

10. Ranked Retrieval Case

In ranked search, precision is often measured at top-k results (e.g., Precision@10).

Recall may be hard to measure at web scale because total relevant documents are unknown.

11. Exam-Ready Definitions

Precision is the fraction of retrieved documents that are relevant to the query.

Recall is the fraction of relevant documents in the collection that are successfully retrieved by the system.
