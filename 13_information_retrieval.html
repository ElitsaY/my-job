1. What Is a K-Gram?

A k-gram is a substring of length k taken from a word.

  • If k = 2, we call them bigrams.
  • If k = 3, we call them trigrams.

To improve matching, we usually add boundary symbols $ at the beginning and end of the word.

Example: word = "cat"
$cat$

Bigrams (k = 2):
$c
ca
at
t$

Trigrams (k = 3):
$ca
cat
at$

2. What Is a K-Gram Index?

A k-gram index maps each k-gram to the list of words that contain it.

Example dictionary
cat
car
cart
cut
K-gram Words containing it
$ccat, car, cart, cut
cacat, car, cart
arcar, cart
atcat
rtcart
utcut

Instead of searching the whole dictionary, we look up a few k-grams and retrieve candidate words quickly.

3. How It Helps in Spell Checking

Suppose a user types "catt". We want to find similar correct words.

  1. Break the query into k-grams.
  2. Find candidate words using the k-gram index.
  3. Compute similarity with the Jaccard coefficient.
  4. Apply edit distance (Levenshtein) to pick the best final correction.
J(A, B) = |A ∩ B| / |A ∪ B|

Where A is the set of k-grams of the misspelled word, and B is the set of k-grams of a candidate word.

Step 2.4: Apply a Similarity Threshold

We usually filter candidates before applying edit distance. For example, compare "catt" vs "cat":

J = 4 / (5 + 4 - 4) = 4/5 = 0.8

High similarity keeps a candidate; low similarity discards it.

Why This Is Efficient

  • Without k-grams, edit distance must be computed against every word in the dictionary.
  • With k-grams, we reduce to a small candidate set (often tens or hundreds of words).

Implementation View

Candidate generation (pseudo)
candidates = empty_map

for each kgram in query:
    word_list = index[kgram]
    for word in word_list:
        candidates[word] += 1

filter candidates using Jaccard threshold

Choosing K

k value Effect
Small k (2)More matches, larger candidate set
Larger k (3 or 4)Fewer matches, smaller candidate set

5. Why K-Gram Indexes Are Useful

  • Reduce search space.
  • Quickly find similar words.
  • Scale well for large vocabularies.
  • Enable wildcard queries like "ca*".

6. Overall Spell Checking Pipeline

  1. User enters word.
  2. Generate k-grams.
  3. Retrieve candidate words from the k-gram index.
  4. Filter using Jaccard similarity.
  5. Rank using edit distance and return the best correction.

7. Intuition and One-Sentence Summary

Think of k-grams as overlapping pieces of a word. If two words share many small pieces, they are probably similar.

Example overlap
catt
cat

Shared bigrams:
$c
ca
at

In one sentence: A k-gram index is an inverted index over substrings of length k that efficiently finds words similar to a misspelled query for spell correction.
