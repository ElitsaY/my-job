Lab Focus: Cross-validation, Confusion Matrices, and the Bias-Variance Tradeoff.

1. Cross-Validation (K-Fold)

Cross-validation estimates how a model will perform on unseen data. It ensures the model hasn't just memorized the training set (Overfitting).

How it works:
  1. Split data into K equal segments (folds).
  2. Train on K-1 folds; test on the remaining 1.
  3. Repeat K times so every fold acts as the test set once.
  4. Average the results for a stable performance score.
Performance = $\frac{\sum \text{Score}_i}{k}$

2. The Confusion Matrix

A table used to describe the performance of a classification model. It breaks down correct predictions vs. specific types of errors.

TP / TN

True Positive/Negative: The model predicted correctly. Correct hits and correct rejections.

FP (Type I)

False Positive: A "False Alarm." The model predicted positive, but the truth was negative.

FN (Type II)

False Negative: A "Miss." The model predicted negative, but the truth was positive.

3. Precision & Recall (Sensitivity)

Depending on the problem, one metric may be more important than the other. In medicine, missing a sick person (Low Recall) is often worse than a false alarm.

Metric Formula Logic
Sensitivity (Recall) $\frac{TP}{TP + FN}$ Ability to find all positive cases.
Specificity $\frac{TN}{TN + FP}$ Ability to correctly identify negative cases.
Precision $\frac{TP}{TP + FP}$ Accuracy of the positive predictions.

4. Bias vs. Variance Tradeoff

The "Sweet Spot" of Machine Learning is finding a model with enough complexity to learn (Low Bias) but enough simplicity to generalize (Low Variance).

High Bias

Underfitting

The model is too simple (e.g., a straight line for curved data). It misses the underlying pattern.

High Variance

Overfitting

The model is too complex (squiggly). It follows the "noise" in the data too closely.
