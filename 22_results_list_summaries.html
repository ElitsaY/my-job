After ranking documents, search systems present a results list with titles and short summaries.

1. Results Lists

Having ranked the documents matching a query, we wish to present a results list.

Most commonly, the list includes:

  • Document title
  • Short summary (snippet)

2. Titles

The title is typically automatically extracted from document metadata.

Example
Title: Machine Learning Basics
Author: Smith
Body: Machine learning is a subfield of AI...

3. Summaries

What about the summaries?

Search systems generate a short snippet that helps users judge relevance quickly.

Common strategies: extract a query-biased snippet or show a highlighted text window around query terms.

4. Why Do We Need Summaries?

When users see results, they must quickly decide whether a document is relevant. The snippet helps users:

  • Understand context.
  • See where query terms appear.
  • Judge relevance quickly.

Without a snippet, users would have to click every link.

5. Two Types of Summaries

Type Description
Static summaryPrecomputed when the index is built
Dynamic summaryGenerated at query time

6. Static Summaries

These are created during indexing, such as:

  • First 1–2 sentences of the document.
  • Abstract (if available).
  • Meta description tag (for web pages).

Advantages

  • Fast (already stored).
  • No computation at query time.

Disadvantages

  • May not contain query terms.
  • Not tailored to the user query.
Example
Query: neural networks

Snippet: “Welcome to my blog about artificial intelligence...”

7. Dynamic Summaries (Most Common)

Modern search engines generate query-dependent snippets at query time.

How Dynamic Snippets Work

  • Find positions of query terms.
  • Extract a text window around them.
  • Highlight query words.
  • Show the most informative window.
Example
Document: Machine learning is a subfield of AI. It allows systems to learn from data.
Query: learn from data

Snippet: “...systems to learn from data...”

8. How Is the Best Snippet Chosen?

Common sentence-based method:

  • Split document into sentences.
  • Score each sentence.
  • Pick the highest-scoring sentence(s).

Sentence score can depend on:

  • Number of query terms.
  • Proximity of query terms.
  • TF-IDF weight of query terms.
  • Whether terms appear close together.

9. Example of Sentence Scoring

Query
machine learning algorithms
Sentences
1) “Machine learning is powerful.”
2) “Many algorithms are used in data mining.”
3) “Machine learning algorithms improve performance.”

Sentence 3 contains all terms close together, so it is chosen.

10. Snippet Window Approach

Instead of full sentences, some systems choose a fixed-size window:

Window scoring
Window size = 30 words
Slide window across document
Score = Σ TF-IDF(query terms in window)

The best window becomes the snippet.

11. Additional Techniques

  • Highlight query terms.
  • Show ellipses (...) around extracted text.
  • Combine two windows if needed.
  • Avoid cutting words in the middle.

12. Trade-offs

Static Dynamic
FastSlightly slower
Not query-specificQuery-specific
Lower relevanceHigher relevance

Modern systems prefer dynamic snippets.

13. Why Not Show Entire Paragraphs?

  • Users scan quickly.
  • Short snippets reduce cognitive load.
  • Focused context improves decision-making.

14. Big Picture

Ranking determines which documents are relevant.

Snippets help users see why a document is relevant.

15. Exam-Ready Answer

After ranking documents, search systems typically display the document title and a short summary. While titles are extracted from metadata, summaries are usually generated dynamically at query time by selecting text fragments containing query terms, often using sentence scoring or window-based extraction, to provide query-dependent and informative context.
