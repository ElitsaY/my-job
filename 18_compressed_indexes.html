What Is a Compressed Index?

A compressed index is an inverted index where postings lists and dictionary terms are encoded with compression techniques to reduce space and improve speed.

1. Why Do We Compress Indexes?

  • Large collections produce massive indexes (hundreds of GBs or TBs).
  • Disk I/O is expensive.
  • Smaller indexes improve cache performance.

Compression reduces storage and disk transfer time, often speeding up queries despite decompression cost.

2. What Gets Compressed?

Two main components:

  • Dictionary (terms)
  • Postings lists (docIDs, term frequencies, positions)

3. Compressing Postings Lists

Postings lists store sorted docIDs.

Example postings
cat → [3, 10, 25, 100]

Step 1: Gap Encoding

Store differences between consecutive docIDs:

Gap encoding
Original: 3, 10, 25, 100
Gaps:     3, 7, 15, 75

Smaller numbers compress better.

Step 2: Variable-Length Encoding

Use fewer bits for small numbers. Common methods:

  • Variable Byte (VB) coding
  • Gamma coding
  • Delta coding

4. Variable Byte (VB) Encoding Example

Use 7 bits for data and 1 bit to mark the end of a number.

Example
Number 7 in binary: 0000111
Stored in 1 byte

Small gaps use fewer bytes, giving strong compression.

5. Gamma Coding (Concept)

Good for small numbers.

Example: N = 10
Binary(10) = 1010
Length = 4
Unary(4) = 1110
Remove leading 1 → 010
Encoded: 1110 010

6. Why Gap Encoding Works

Sorted docIDs often have small gaps.

Dense IDs
DocIDs: 1001, 1002, 1003, 1004
Gaps:   1001, 1, 1, 1

Small numbers compress extremely well.

7. Compressing Term Frequencies & Positions

Positions in positional indexes are also sorted and gap-encoded:

Example
cat → (doc 3: positions 1, 8, 15)
Gaps: 1, 7, 7

8. Dictionary Compression

Dictionary terms often share prefixes.

Blocking

Example
apple#5apply#5application#11apt#3

Front Coding

Example
apple
2 ly
7 ication

Store prefix once and encode differences.

9. Overall Compression Structure

Compressed dictionary + compressed postings lists

10. Why Compression Improves Speed

  • Disk I/O is slower than CPU.
  • Smaller indexes mean fewer disk reads.
  • Better cache locality.

Net effect: faster query processing despite decompression cost.

11. Example Before vs After Compression

Original postings
[1000, 1050, 1100, 1200]
Stored as 4 integers = 16 bytes
Gap + VB (example)
Gaps: [1000, 50, 50, 100]
Encoded size: ~6–8 bytes

At web scale, savings are huge.

12. Tradeoff

Advantage Disadvantage
Smaller indexDecompression cost
Faster disk accessMore complex code
Better cache usageSlight CPU overhead

13. Key Insight

  • Postings lists are sorted.
  • Gaps are small.
  • Natural language has redundancy.

14. Exam-Ready Definition

A compressed index is an inverted index where postings lists and dictionary entries are encoded using techniques such as gap encoding and variable-length coding to reduce storage space and improve query performance.
