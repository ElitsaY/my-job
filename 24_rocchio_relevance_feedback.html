What Problem Does Rocchio Solve?

After a user submits a query, the system retrieves documents. Some are relevant, some are not. The user provides feedback:

  • ✔ Relevant documents
  • ✘ Non-relevant documents

Rocchio tells us how to modify the query to get better results next time.

1. Core Idea

Move the query vector:

  • Toward relevant documents
  • Away from non-relevant documents

2. Rocchio Formula

q_new = α q_old + β (1/|Dr|) Σ d_r − γ (1/|Dnr|) Σ d_nr
  • q_old = original query vector
  • Dr = set of relevant documents
  • Dnr = set of non-relevant documents
  • α = weight for original query
  • β = weight for relevant docs
  • γ = weight for non-relevant docs

3. Intuition

Relevant documents tend to cluster in one region of vector space, non-relevant documents cluster elsewhere. Rocchio moves the query toward the relevant cluster and away from the non-relevant cluster.

4. Simple Numerical Example

Vocabulary: machine, learning

Original query
q = (1, 0)
Relevant docs
d1 = (1, 1)
d2 = (2, 1)
Average relevant vector
((1,1) + (2,1)) / 2 = (1.5, 1)
Non-relevant doc
d3 = (0, 2)

Assume parameters: α=1, β=0.75, γ=0.25

Compute
Relevant part: 0.75 × (1.5, 1) = (1.125, 0.75)
Non-relevant part: 0.25 × (0, 2) = (0, 0.5)

q_new = (1, 0) + (1.125, 0.75) − (0, 0.5)
q_new = (2.125, 0.25)

Result: the query gains weight on learning and strengthens machine.

5. Why This Works

Relevant documents often contain useful additional terms. Rocchio learns and adds those terms automatically.

6. Pseudo-Relevance Feedback

When users do not provide feedback, systems often assume top-k documents are relevant and apply Rocchio automatically.

7. Typical Parameter Values

  • α = 1
  • β = 0.75
  • γ = 0.15

8. Advantages

  • Simple and efficient.
  • Improves recall.
  • Automatically expands queries.

9. Limitations

  • Assumes linear vector space.
  • Requires relevance judgments.
  • Can drift if feedback is wrong.

10. Geometric View

Before feedback, the query is between clusters. After feedback, it moves toward the relevant cluster center.

11. Exam-Ready Definition

The Rocchio Algorithm is a relevance feedback method in the Vector Space Model that modifies a query vector by moving it toward the centroid of relevant documents and away from non-relevant documents using weighted vector adjustment.

12. Relevance Feedback in Vector Spaces

Relevance feedback improves retrieval by updating the query vector based on user-identified relevant and non-relevant documents. The process:

  • Retrieve results for the initial query.
  • User marks relevant and non-relevant documents.
  • Compute centroids of relevant and non-relevant sets.
  • Update query using Rocchio.
  • Re-rank documents with the new query.

Types of relevance feedback include:

  • Explicit feedback (user labels)
  • Pseudo-relevance feedback (assume top-k relevant)
  • Implicit feedback (clicks, dwell time)

13. Probabilistic Relevance Feedback

Probabilistic feedback re-estimates how likely each term appears in relevant vs non-relevant documents under the Binary Independence Model (BIM).

w_t = log((P(t|R=1)(1−P(t|R=0))) / (P(t|R=0)(1−P(t|R=1))))

Estimate probabilities from relevance judgments:

P(t|R=1) ≈ r_t / R
P(t|R=0) ≈ (n_t − r_t) / (N − R)

Intuition: terms that appear often in relevant docs but rarely in non-relevant docs get higher weights.

Numerical Example

Given
N = 100, R = 10
Term "machine": n_t = 20, r_t = 8
Probabilities
P(t|R=1) = 8/10 = 0.8
P(t|R=0) = (20-8)/(100-10) = 12/90 ≈ 0.133
Weight
w_t = log( (0.8 × 0.867) / (0.133 × 0.2) )
    ≈ log(26)
