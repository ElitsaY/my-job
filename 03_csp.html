Constraint Satisfaction Problems

← Back

Lab Focus: Identification problems, variable constraints, and local search heuristics.

1. CSP Fundamentals

CSPs differ from standard search because they are identification problems—we only care about the final state satisfying all rules, not the path taken.

Variables (X)

N variables that need values assigned.

Domain (D)

Set of allowable values for each variable.

Constraints (C)

Restrictions on variable combinations.

2. Constraint Types

  • Unary: Restricts a single variable (e.g., SA ≠ Green).
  • Binary: Relates two variables (e.g., SA ≠ WA). Represented as edges in a graph.
  • Higher-Order: Involves 3 or more variables (e.g., Room capacity constraints in timetabling).
  • Soft Constraints: Represent preferences rather than hard rules.

3. Solving Techniques

Backtracking (DFS Optimization)

Backtracking improves basic DFS by fixing variable ordering and checking constraints before moving to the next level.

MRV Heuristic

Choose the "Most Constrained Variable" (one with fewest remaining valid values) first.

LCV Heuristic

Choose the "Least Constraining Value"—the one that leaves the most options for neighbors.

Arc Consistency (AC-3) Insights

The AC-3 algorithm simplifies problems by pruning variable domains using binary constraints.

  • Every binary constraint becomes two directed arcs [00:00:30].
  • An arc (X → Y) is consistent if every value in X has at least one valid partner in Y [00:01:23].
  • If a domain is pruned, all arcs pointing to that variable must be re-added to the agenda for re-checking [00:02:06].

4. Local Search: Min-Conflicts

Min-Conflicts is a hill-climbing variant that starts with a random complete assignment and iteratively repairs it.

h(n) = Total number of constraint violations (conflicts)

The N-Queens Strategy

  1. Initialize all queens randomly (each in a separate column).
  2. Pick a random conflicted queen.
  3. Move her to the row in her column that results in the minimum conflicts.
  4. Repeat until h(n) = 0.
