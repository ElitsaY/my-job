K-Nearest Neighbors (KNN)

← Back

Lab Focus: Comparison of learning paradigms and the intuition behind KNN.

1. Learning Paradigms

Before diving into KNN, we must distinguish how different algorithms process and store information.

Scope of Learning

Global vs. Local

Global: Generalizes from the entire dataset (e.g., Linear Regression).

Local: Focuses only on specific nearby instances (e.g., KNN).

Method of Storage

Instance vs. Model

Instance-Based: The data is the model. No abstraction is created.

Model-Based: Extracts parameters (like weights) and discards raw data after training.

Execution Timing

Lazy vs. Eager

Lazy: No work is done until a query is made. "Store now, calculate later."

Eager: Pre-calculates a model before receiving test data (e.g., SVM, Neural Networks).

2. How KNN Works

The core philosophy: "Tell me who your neighbors are, and I'll tell you who you are."

The Process:
  1. Store all training data.
  2. When new data arrives, calculate the distance (e.g., Euclidean) to all stored points.
  3. Find the K closest points.
  4. Classification: Take the majority vote.
  5. Regression: Take the average of the neighbors.

3. Pros & Cons

Advantages

  • Intuitive: Extremely easy to explain.
  • Zero Training Time: No complex mathematical fitting.
  • Non-Parametric: No assumptions about data shape/distribution.
  • Versatile: Handles both classes and numbers.

Disadvantages

  • Computationally Expensive: Slow testing phase (must check every point).
  • Memory Heavy: Requires storing the entire dataset.
  • Scale Sensitive: Distances are ruined if features aren't normalized.
  • Outlier Sensitive: Small K values can be misled by noise.

4. Distance Metrics

Since KNN relies on "closeness," the definition of distance is critical:

Euclidean: d = √Σ(xi - yi)^2
Manhattan: d = Σ|xi - yi|
Minkowski: d = (Σ|xi - yi|^p)^(1/p)
