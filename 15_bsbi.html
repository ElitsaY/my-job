What Is BSBI?

Blocked Sort-Based Indexing (BSBI) builds an inverted index efficiently for very large collections by using external sorting and disk-based merging.

1. Why Do We Need BSBI?

When building an inverted index, we generate many (term, docID) pairs.

  • Large collections create millions of pairs.
  • All pairs cannot fit in RAM.
  • We cannot sort everything at once in memory.

BSBI solves this using external sorting and block-based processing.

2. Main Idea of BSBI

Divide → Sort → Write → Merge
Split documents into blocks
For each block:
    Generate (termID, docID)
    Sort in memory
    Write partial index to disk
Merge all partial indexes

3. Step-by-Step Algorithm

Step 1: Split Collection into Blocks

If memory can handle 100,000 pairs and the collection generates 10 million pairs:

10,000,000 / 100,000 = 100 blocks

Step 2: Process Each Block

(a) Parse documents and extract tokens

Example documents
Doc1: "cat dog"
Doc2: "dog fish"

Generate term-doc pairs:

Pairs
(cat, 1)
(dog, 1)
(dog, 2)
(fish, 2)

(b) Convert terms to termIDs

Term IDs
cat  → 1
dog  → 2
fish → 3

Pairs become:

TermID pairs
(1, 1)
(2, 1)
(2, 2)
(3, 2)

(c) Sort pairs by termID, then docID

Sorted
(1, 1)
(2, 1)
(2, 2)
(3, 2)

(d) Write partial inverted index to disk

Postings lists
1 → [1]
2 → [1, 2]
3 → [2]

Repeat for all blocks.

4. Final Step: Merge Blocks

Merge partial indexes using multi-way merge (like merge sort).

Example merge
Block 1:
cat → [1, 4]
dog → [1]

Block 2:
cat → [7]
fish → [2]

Final:
cat → [1, 4, 7]
dog → [1]
fish → [2]

5. Why Sorting Is Important

  • Groups the same terms together.
  • Makes postings list creation easy.
  • Enables efficient merging across blocks.

6. Complexity

Let T be total term-doc pairs and B be block size.

Time complexity ≈ O(T log T)

Sorting happens within blocks and during merge. Disk I/O dominates cost in practice.

8. Why BSBI Is Efficient

  • Works when the index does not fit in RAM.
  • Uses sequential disk access (fast).
  • Relies on external merge sort.
  • Scales to web-size collections.

9. BSBI vs SPIMI

Feature BSBI SPIMI
Needs global sortingYesNo
Uses termID mappingYesNo
Memory usageEfficientMore flexible
SpeedSlightly slowerFaster

SPIMI avoids global sorting but uses hash tables.

10. Key Insight

BSBI is essentially external merge sort applied to inverted index construction.

11. Exam Definition

Blocked Sort-Based Indexing (BSBI) is an external memory indexing algorithm that divides a document collection into memory-sized blocks, sorts term–document pairs within each block, writes partial inverted indexes to disk, and then merges them to create a final inverted index.
