Lab 02: Informed Search

← Back

Recap on search evaluation, Heuristics, Greedy, A*, and Optimization techniques.

1. Recap & Definitions

Informed Search strategies use problem-specific knowledge (heuristics) to find solutions more efficiently than uninformed "blind" search.

Search Dimensions

  • Completeness Always finds a solution if one exists.
  • Optimality Finds the least-cost solution.
  • Time Complexity Number of nodes generated/expanded.
  • Space Complexity Maximum nodes stored in memory.

2. Heuristics

A heuristic function h(n) estimates the cost from state n to the goal. It guides the search to prefer "promising" states.

  • Admissible: Never overestimates the true cost to the goal (h(n) ≤ h*(n)).
  • Consistent: The estimated cost from n to goal is no greater than the step cost to a neighbor n' plus the estimated cost from n' to goal.

3. Greedy Search

Always expands the node that appears closest to the goal (lowest h(n)).

  • Optimal: No.
  • Complete: No (can get stuck in loops).
  • Complexity: O(bm) for both time and space.

4. A* Search

Combines the cost to reach the node g(n) and the estimated cost to the goal h(n).

f(n) = g(n) + h(n)
  • Optimality: Guaranteed if the heuristic is admissible (for Tree Search) or consistent (for Graph Search).
  • Efficiency: A* is optimally efficient—no other optimal algorithm expands fewer nodes.

5. Optimization Algorithms

Used when the path doesn't matter, only the final goal state (e.g., 8-Queens, NLP).

  • Beam Search: Keeps only the top l most promising nodes at each level.
  • Hill Climbing: Keeps only the single best successor (Beam Search where l=1). Can get stuck in local maxima.
  • Simulated Annealing: Escapes local maxima by allowing "bad moves" with a probability that decreases as the "temperature" cools down.

6. Summary of Search Strategies

Algorithm Complete Optimal Time Space
Greedy No* No O(bm) O(bm)
Beam Search No No O(bml) O(bl)
Hill Climbing No No O(bm) O(b)
A* Yes* Yes O(bd) O(bd)
IDA* Yes Yes O(bd) O(bd)

7. Video Resource: Simulated Annealing

The core idea of Simulated Annealing is based on a metallurgical process. It explores widely when "hot" and settles into an optimal state as it "cools".

Key Takeaways from Video:

  • Hill climbing gets stuck at local maxima [00:00:52].
  • Stochastic Hill Climbing picks random uphill moves to avoid getting stuck [00:02:35].
  • Simulated Annealing allows "bad moves" based on probability: eΔE / T [00:07:53].
  • As Temperature (T) decreases, the algorithm becomes more selective, eventually settling on a global maximum [00:09:45].
