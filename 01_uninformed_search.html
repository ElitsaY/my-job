Lab 01: Basic Definitions & Uninformed Search

Explore basic concepts of AI: State, State Space, and Uninformed Search strategies.

Basic Concepts and Definitions

State Space

  • State: A representation of the task in the process of its solution.
    • Initial State Represented with S
    • Intermediate State Any capital letter except S and G
    • Goal State Represented with G (G1, G2, etc.)
  • Successor Function: Obtaining one state from another.
  • Path Cost: Additive sum of distances or actions.
  • State Space: Totality of all possible states obtained from the initial state.
"A solution is a sequence of actions leading from the initial state to a goal state."

Representation of the State Space

The state space is represented as a graph or a tree, where each state is a node and the successor function is an edge.

bMaximum branching factor of the search tree
dDepth of the least-cost solution
mMaximum depth of the state space (may be ∞)

DFS - Depth First Search

Go Deep First: DFS explores as far as possible down one branch before backtracking. It's like exploring a maze by always choosing the first available path until hitting a dead end.

  • Data Structure: Stack (LIFO). Pops the most recent node to backtrack.
  • Complete: No Fails in infinite-depth spaces.
  • Optimal: No Might find a deep solution when a shallow one exists.
Infinite Space Trap Example:
Target 10 from Start 5 using "Multiply by 2" or "Subtract 1".
DFS might go: 5 → 10 → 20 → 40 → ... ∞ (missing the target at the first step).
Time Complexity: O(bm)
Calculated as: b0 + b1 + ... + bm
Space Complexity: O(bm)

BFS - Breadth First Search

Explores all nodes at the current depth level before moving to the next level.

  • Data Structure: Queue (FIFO).
  • Complete: Yes If branching factor b is finite.
  • Optimal: Yes Only if all step costs are equal.
Time Complexity: O(bd+1)
Explores all nodes up to d, and generates children for level d.
Space Complexity: O(bd+1)

Uniform-Cost Search (UCS)

Selects the node on the frontier with the lowest path cost from the start node. This is essentially Dijkstra's algorithm applied to search.

  • Frontier: Priority Queue (Heap-based).
  • Complete: Yes If step costs ε > 0.
  • Optimal: Yes Guaranteed to find the lowest-cost path.
Let C* be the optimal cost and ε be the minimum step cost.
Time Complexity: O(b⌈ C*/ε ⌉)
Space Complexity: O(b⌈ C*/ε ⌉)

Depth-Limited Search (DLS)

A variation of DFS that addresses infinite paths by supplying a limit l. Nodes at depth l have no successors.

CompleteYes, if l ≥ d
OptimalNo
Time ComplexityO(bl)
Space ComplexityO(bl)

Iterative Deepening Search (IDS)

IDS repeatedly applies DLS, increasing the limit l until a goal is found. It explores level-by-level like BFS but uses DFS memory efficiency.

Time Complexity: O(bd)
(d+1)b0 + db1 + (d-1)b2 + ... + 1bd
Example: b = 10, d = 5
• NIDS = 123,456
• NBFS = 1,111,101

Summary of Uninformed Algorithms

Algorithm Complete Optimal Time Space Structure
DFS No No O(bm) O(bm) Stack
BFS Yes* Yes* O(bd+1) O(bd+1) Queue
UCS Yes Yes O(b⌈ C*/ε ⌉) O(b⌈ C*/ε ⌉) Priority Queue
DLS Yes* No O(bl) O(bl) Stack
IDS Yes Yes O(bd) O(bd) Stack

*Subject to conditions (finite b, unit step costs, or l ≥ d).
